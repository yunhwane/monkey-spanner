# 다형성 = "분기를 한 곳으로 모으기", OCP를 스스로 검증함

Lesson 03 실습에서 Notifier 인터페이스 설계 후, 두 핵심 질문에 정확히 답함:
(1) "분기는 없어진 게 아니라 팩토리 한 곳으로 모인다", (2) "새 채널은 클래스 하나 추가로 끝, 기존 코드 수정 0(OCP)".
암기가 아니라 자기 언어로 다형성의 본질과 개방-폐쇄 원칙을 연결 — 면접 목표에 직접 부합.

## Evidence
- interface Notifier { void send(User, msg) } 정확히 설계
- 분기가 "제거"가 아니라 "생성 지점으로 이동"임을 스스로 표현 (핵심 통찰)
- 확장 비용을 파일 수 기준으로 정량화 (신규 1, 수정 0)
- 추가로 Map/DI로 팩토리 분기까지 제거하는 확장은 교사가 제시함(아직 스스로 도달 X)

## Implications
- OOP 4축 중 다형성 + SOLID의 O 완료. 남은 축: 상속(가장 오해 많은 축).
- 다음: Lesson 04 상속 vs 조합 + LSP. "상속을 언제 쓰고 언제 독이 되는가".
- 이후: SOLID 나머지(SRP/ISP/DIP)를 지금까지 예제 위에서 통합, 그 다음 오픈소스 분석 세션(사용자 희망).
- 학습자는 개념→코드 전이가 빠름. 설명 압축하고 실습/질문 비중 유지.
