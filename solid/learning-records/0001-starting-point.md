# 시작점: Java 학습자, 면접 대비, OOP는 "외운 지식" 상태

세션 1에서 확립된 배경. 사용자는 Java가 주력이며 목표는 취업/면접 대비다. OOP(캡슐화·추상화·상속·다형성)와
SOLID를 개념으로는 배웠으나 **코드 레벨에서 꺼내 쓰지 못한다** — 암기와 적용 사이의 간극이 핵심 과제다.

## Implications
- 모든 개념은 "정의 암기"가 아니라 "지저분한 Java 코드 → 리팩토링 → 원칙 이름 붙이기" 흐름으로 가르친다.
- ZPD: 문법은 알지만 설계 안목은 초급. 문법 설명은 건너뛰고 설계 판단에 집중.
- 강의의 뿌리 서사는 사용자가 정한 것: **추상 → 클린코드 → OOP → SOLID → 오픈소스 분석**.
- Lesson 01에서 "한 함수 한 수준의 추상화 / Extract Method"를 다룸. 다음 후보: 캡슐화(객체 뒤로 데이터+로직 감추기).
