# 잘 읽히는 코드 (클린코드 & OOP, Java) — Resources

## Knowledge

- [책: _Clean Code_ — Robert C. Martin](https://www.oreilly.com/library/view/clean-code-a/9780136083238/)
  이 강의의 근간. 특히 Ch.2(의미 있는 이름), Ch.3(함수 — "한 함수는 한 수준의 추상화"), Ch.6(객체와 자료구조). 원칙의 "왜"를 찾을 때.
- [Clean Code 핵심 요약 — wojteklu gist](https://gist.github.com/wojteklu/73c6914cc446146b8b533c0988cf8d29)
  책 전체 원칙을 한 페이지로 압축. 빠른 복습·용어 확인용.
- [Refactoring Guru — Code Smells 카탈로그](https://refactoring.guru/refactoring/smells)
  21가지 코드 냄새 + 66가지 리팩토링(Java 예제 포함). "이 코드의 무엇이 문제인가" 이름 붙일 때.
- [Refactoring Guru — Refactoring 카탈로그](https://refactoring.guru/refactoring/catalog)
  Extract Method 등 리팩토링 기법의 표준 이름·절차. 실습 단계 설계의 근거.
- [Baeldung — A Solid Guide to SOLID Principles](https://www.baeldung.com/solid-principles)
  SOLID 5원칙을 Java 코드로 하나씩. 면접 답변의 근거·예시.
- [책: _Clean Architecture_ — Robert C. Martin](https://www.oreilly.com/library/view/clean-architecture/9780134494272/)
  SOLID와 "의존성의 방향"을 시스템 규모로 확장. DIP/경계 설계의 뿌리. SOLID를 "왜"까지 이해할 때.
- [Uncle Bob — Clean Code 강의 영상 (cleancoders.com)](https://cleancoders.com/)
  저자 본인의 SOLID/함수 설계 강의. 개념의 원출처를 영상으로 볼 때.
- [책: _Effective Java_ (3rd ed.) — Joshua Bloch](https://www.oreilly.com/library/view/effective-java/9780134686097/)
  Java 설계의 최고 권위서. Item 18(상속보다 조합), Item 17(불변), Item 20(인터페이스) 등. 상속·조합·API 설계 판단의 근거.

## Wisdom (Communities)

- [OKKY 커뮤니티](https://okky.kr/)
  국내 개발자 Q&A/취업 게시판. 코드 리뷰 요청·면접 후기·설계 토론. 한국어로 실전 피드백 받기 좋음.
- [r/javahelp](https://reddit.com/r/javahelp) · [r/java](https://reddit.com/r/java)
  Java 코드 리뷰와 설계 질문. 영어. 리팩토링 결과물 피드백용.
- [Exercism — Java Track (멘토링)](https://exercism.org/tracks/java)
  실제 사람 멘토가 코드에 리뷰를 달아줌. 리팩토링 습관을 외부 피드백으로 검증할 때.
