# NOTES — 학습자 선호 & 작업 메모

## 학습자
- 주력 언어: **Java**
- 목표: 취업/면접 대비. OOP/SOLID를 "외운 지식"에서 "코드로 쓰는 능력"으로 전환하고 싶어 함
- 세션 분량: **30분 내외** — 개념 + 짧은 리팩토링 실습이 한 호흡
- 배경: 동국대(dgu.ac.kr) 재학 추정. OOP 개념은 배웠으나 코드 레벨 적용이 약함

## 교육 방침 (이 강의의 철학)
- **추상(抽象)을 뿌리로** 삼아 클린코드·OOP·SOLID를 하나의 나무로 연결한다 (사용자가 붙여준 강의 설명의 핵심)
- 법칙을 맹목적으로 암기시키지 말고, "왜"를 먼저 납득시킨 뒤 리팩토링으로 체득
- 매 레슨: 지저분한 코드 → 단계적 리팩토링 → 원칙 이름 붙이기
- 뒤로 갈수록 실제 오픈소스 코드 분석 세션 포함 (사용자가 원함)

## 작업 메모
- 레슨은 Tufte 스타일 공유 스타일시트(assets/style.css) 사용
- 퀴즈는 assets/quiz.js 재사용. 보기는 글자 수 최대한 균등하게
