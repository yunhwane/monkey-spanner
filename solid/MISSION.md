# Mission: 잘 읽히는 코드 — 추상에서 출발하는 클린코드 & 객체지향 (Java)

## Why
"와, 이 사람 코드 잘 짠다"는 말은 결국 "잘 읽힌다"는 뜻이다. 사용자는 OOP·SOLID·클린코드를
면접용으로 외우기만 했고, 코드 레벨에서 즉시 꺼내 쓰지 못한다. 목표는 이 개념들을 **말로 설명하고
실제 Java 코드로 적용·리팩토링**할 수 있게 되어, 취업/면접에서 "왜 이렇게 짰는가"를 근거와 함께
답할 수 있는 상태가 되는 것.

## Success looks like
- 지저분한 Java 코드를 보고 어떤 원칙이 깨졌는지 이름 붙여 지적하고, 단계적으로 리팩토링할 수 있다
- 캡슐화·추상화·상속·다형성과 SOLID 5원칙을 각각 "한 문장 정의 + Java 코드 예시 + 왜 필요한가"로 설명할 수 있다
- 실제 오픈소스(예: Spring, JDK)의 코드에서 객체지향 설계 의도를 읽어내고 말로 풀어낼 수 있다
- 면접에서 "추상화가 뭔가요?" 같은 질문에 암기가 아니라 자기 언어로 답한다

## Constraints
- 주력 언어: **Java** (모든 코드 예제는 Java)
- 세션당 30분 내외 — 개념 + 짧은 리팩토링 실습이 한 호흡에 들어가야 함
- 목표: 취업/면접 대비 (학생, dgu.ac.kr)

## Out of scope
- 특정 프레임워크 심화(Spring 전체, JPA 등) — 설계 원칙을 보기 위한 소재로만 등장
- 함수형 프로그래밍, 동시성 등 OOP 밖의 패러다임
- 빌드 도구/배포/인프라
