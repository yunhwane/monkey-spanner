# Mission: Loop Engineering (에이전트 루프 설계)

## Why
직접 동작하는 AI 에이전트/자동화를 스스로 만들 수 있게 되는 것. 한 번의 프롬프트가 아니라,
스스로 행동하고 결과를 검증하며 목표에 도달할 때까지 반복하는 "루프"를 설계할 수 있게 된다.

## Success looks like
- 작동하는 최소 에이전트 루프(gather → act → verify → repeat)를 직접 코드로 짤 수 있다
- 어떤 작업에 대해 "검증 신호(verification signal)"가 무엇인지 식별하고 루프에 연결할 수 있다
- 무한루프/헛도는 루프를 막는 종료 조건과 가드레일을 설계할 수 있다
- 자기 업무 중 반복 작업 하나를 골라 실제로 자동화 루프로 만든다

## Constraints
- 학습 방식: **직접 만들며 배우기** (작은 코드/실습 위주, 읽기만 X)
- 현재 수준: Claude Code 등 에이전트 도구는 써봤으나 루프를 직접 구현해본 적은 없음
- 언어: 한국어로 설명

## Out of scope (지금은)
- 멀티 에이전트 오케스트레이션 (먼저 단일 루프부터)
- 모델 파인튜닝 / 학습
- 프로덕션 배포·인프라 (개념과 구현이 먼저)
