Loop Engineering · Lesson 03

종료 조건 & 가드레일 — 멈출 줄 아는 루프

Lesson 02에서 루프는 객관적 신호로 "성공하면" 멈춘다고 배웠습니다. 그런데 현실의 에이전트는 자주 성공하지 못합니다. 막히고, 같은 실수를 반복하고, 빙빙 돕니다. 오늘 끝나면 당신은 헛도는 루프를 자동으로 감지해 조기에 멈추는 가드레일을 설계할 수 있습니다.

먼저 떠올려 보기 (1초)

Lesson 01 코드의 range(10)은 어떤 가드레일이었죠? … 최대 반복 횟수. 오늘은 그것만으로는 왜 부족한지, 뭘 더해야 하는지를 봅니다.

루프가 멈추는 길은 둘뿐

① 성공 종료 객관적 신호 도달 verify() == True ◀ Lesson 02 ② 포기 종료 가드레일 발동 더 해봐야 소용없음 ◀ 오늘 둘 중 하나도 없으면 루프는 영원히 돈다 (또는 비용을 태운다).

가드레일 세 종류

  • 최대 반복 (max steps) — 절대 상한. 안전망이지 전략은 아님.
  • 무진전 감지 (no-progress) — "N회 연속 나아지지 않으면 중단". ★ 오늘의 핵심.
  • 예산 한도 (budget) — 토큰/시간/비용 상한. 실제 LLM 루프에서 필수.
오늘의 핵심 한 가지

max-steps만 두면 막힌 루프가 끝까지 헛돈다. 50번 상한인데 5번째에 이미 막혔다면 45번을 낭비합니다. 진전(progress)을 점수로 재서, 안 오르면 일찍 포기하게 만드세요.

직접 만들어 보기 — 무진전 감지

아래 루프의 model()은 일부러 5에서 막힙니다(목표는 8). max-steps는 50이지만, 무진전 가드레일이 3회째에 조기 종료시켜 낭비를 막습니다. 그대로 실행: python3 guard.py.

def model(state):
    return min(state["v"] + 1, 5)     # 일부러 5에 막히는 가짜 에이전트 (목표=8)

def score(candidate):                 # 진전을 '점수'로 잰다 (pass/fail보다 풍부한 신호)
    return candidate                  # 높을수록 목표에 가까움

state, best, stale = {"v": 0}, -1, 0
for step in range(50):                         # 가드레일 ①: 절대 상한
    cand = model(state)
    s = score(cand)
    print(f"step {step}: score {s}  (best {best}, stale {stale})")
    if s >= 8:
        print("✅ 성공 종료"); break
    if s > best:
        best, stale = s, 0                     # 진전 있음 → 카운터 리셋
    else:
        stale += 1                             # 진전 없음 → 누적
    if stale >= 3:                             # 가드레일 ②: 무진전 감지
        print(f"⛔ 3회 연속 무진전 (best={best}) — 조기 포기"); break
    state["v"] = cand

실행하면 50번이 아니라 대략 8스텝 근처에서 멈춥니다. 막힌 걸 루프가 스스로 알아챈 거죠.
