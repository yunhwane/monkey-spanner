Loop Engineering · Lesson 04

model() 자리에 진짜 에이전트 — Claude Code CLI

여태 model()은 state["v"] + 1 같은 장난감이었습니다. 오늘 그 자리에 진짜 에이전트를 끼웁니다. 그리고 가장 게으른(=효율적인) 방법은 API SDK가 아니라 — 당신이 이미 쓰는 claude CLI를 그대로 호출하는 거예요. 끝나면 당신은 첫 진짜 에이전트 루프를 손에 쥡니다.

먼저 알아야 할 한 가지

Claude Code 자체가 이미 루프다. 파일을 찾고(gather) → 고치고(act) → 테스트 돌려보는(verify) 걸 내부에서 합니다. 그래서 우리가 만드는 바깥 루프는 그 에이전트를 한 단계로 호출하고, verify와 가드레일은 우리가 직접 통제합니다. Lesson 01–03이 전부 여기서 쓰입니다.

먼저 30초: 헤드리스 모드 확인

대화창 없이 한 번 불러봅니다. 터미널에:

claude -p "2 더하기 2는?"

답만 출력하고 끝나죠? -p(=--print)가 헤드리스 모드입니다 — 스크립트에서 부를 수 있는 형태. 이게 우리 루프의 model()이 됩니다.

핵심 플래그 (외울 건 네 개뿐)

  • -p "프롬프트" — 헤드리스 실행, 출력 후 종료
  • --output-format json — 결과+메타데이터를 JSON으로 (파싱하기 좋음)
  • --allowedTools Edit Read — 에이전트가 쓸 수 있는 도구 화이트리스트
  • --permission-mode acceptEdits — 파일 편집은 자동 승인(but 위험한 건 막힘)
⚠️ 보안 — 게을러도 여기선 게으르지 말 것

--dangerously-skip-permissions 는 모든 안전장치를 끕니다. 스크립트에선 대신 --allowedTools 로 도구를 좁히고 --permission-mode acceptEdits 를 쓰세요. 그리고 아래 실습은 파일을 실제로 수정하고 API 호출을 실제로 합니다 — 반드시 빈 연습용 폴더에서 돌리세요.

직접 만들어 보기 — 첫 진짜 코딩 에이전트

"테스트가 통과할 때까지 Claude가 함수를 고치는" 루프입니다. 배운 네 박자가 전부 들어있어요. 빈 폴더를 만들고 아래 두 파일을 두세요.

1) 연습용 폴더 + 테스트 파일

mkdir agent-demo && cd agent-demo

# test_add.py  (검증 신호의 원천)
cat > test_add.py <<'PY'
from solution import add
def test_add():
    assert add(2, 3) == 5
    assert add(-1, 1) == 0
PY

# solution.py  (일부러 비워둠 — 에이전트가 채울 것)
echo "def add(a, b):\n    pass" > solution.py

2) loop.py — 에이전트 루프

import subprocess

def claude(prompt):                          # ★ model() = 진짜 에이전트
    subprocess.run([
        "claude", "-p", prompt,
        "--allowedTools", "Edit", "Read",
        "--permission-mode", "acceptEdits",
    ])

def verify():                                # VERIFY = 객관적 신호 (Lesson 02)
    r = subprocess.run(["python3", "-m", "pytest", "-q"])
    return r.returncode == 0                 # exit 0 = 통과

msg = "solution.py의 add() 함수를 구현해서 test_add.py가 통과하게 해줘."
for step in range(5):                         # 가드레일 (Lesson 03)
    print(f"\n=== step {step} ===")
    claude(msg)                               # GATHER + ACT (에이전트가 내부에서 수행)
    if verify():                              # VERIFY
        print("✅ 테스트 통과 — 종료"); break
    msg = "테스트가 아직 실패해. pytest 출력을 보고 add()를 고쳐줘."  # 피드백 (Lesson 01)
else:
    print("⛔ 5회 내 실패 — 가드레일 작동")

python3 loop.py 로 실행하세요. Claude가 solution.py를 직접 고치고, 당신의 루프가 pytest로 스스로 검증한 뒤, 통과하면 멈춥니다. — 첫 진짜 에이전트 완성. 🎉

방금 일어난 일 (네 박자 매핑)
model() = claude CLI 호출 ← 진짜 에이전트가 GATHER+ACT verify() = pytest exit code ← 객관적 신호 (Lesson 02) range(5) = 최대 반복 ← 가드레일 (Lesson 03) msg 갱신 = 실패 출력 → 다음 회차 ← 피드백 (Lesson 01)
