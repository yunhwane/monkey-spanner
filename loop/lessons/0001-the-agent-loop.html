Loop Engineering · Lesson 01

에이전트 루프, 가장 작은 것부터

당신의 목표는 직접 동작하는 에이전트를 만드는 것입니다. 그 모든 것의 심장은 딱 한 가지 구조 — 루프 — 입니다. 이 레슨이 끝나면 당신은 실제로 돌아가는 최소 에이전트 루프를 손으로 짜고, 왜 그게 에이전트인지 설명할 수 있게 됩니다.

한 줄 정의

에이전트란 LLM이 도구(tool)를 루프 안에서 자율적으로 사용하는 것이다. 프롬프트는 1번 묻고 끝나지만, 에이전트는 스스로 다음 행동을 정하며 반복한다. — Anthropic, Building Effective Agents

루프의 네 박자

거의 모든 에이전트는 이 네 단계를 반복합니다. Claude Code도 내부에서 이렇게 돕니다.

┌──────────────────────────────────────────────┐ │ 1. GATHER 지금 상태가 뭐지? (컨텍스트 수집) │ │ 2. ACT 한 걸음 행동한다 (도구 호출) │ │ 3. VERIFY 진짜 됐나? (검증 신호 확인) │ ◀── 핵심 │ 4. REPEAT 목표 달성? 아니면 1번으로 │ └────────────────────┘ 아니면 위로 ◀────────────┘
이번 레슨의 핵심 한 가지

VERIFY가 루프를 에이전트로 만든다. 검증 신호(테스트 통과 여부, 컴파일 결과, API 응답)가 없으면 모델은 "다 된 것 같아요"라고 말하고 멈출 뿐입니다. 객관적 검증 신호가 있어야 루프가 스스로 틀린 걸 알고 다시 돕니다.

직접 만들어 보기 — 15줄 루프

아래는 진짜로 돌아가는 최소 루프입니다. "비밀번호를 맞출 때까지 추측" 하는 장난감 에이전트인데, 네 박자가 전부 들어 있습니다. API 키 없이 그대로 실행돼요. 파일로 저장하고 python3 loop.py 로 실행해 보세요.

def model(state):           # ← 나중에 진짜 LLM 호출로 교체할 자리
    guess = state["last"] + 1
    return guess

def verify(guess):          # 3. VERIFY — 객관적 검증 신호
    return guess == 7       # (현실에선: 테스트 통과? 컴파일 성공?)

state = {"last": 0}
for step in range(10):                  # 4. REPEAT (+ 무한루프 가드: 최대 10회)
    guess = model(state)                # 1+2. GATHER 상태 → ACT 행동
    print(f"step {step}: tried {guess}")
    if verify(guess):                   # 3. VERIFY
        print("✅ 목표 달성, 루프 종료")
        break
    state["last"] = guess               # 피드백을 다음 회차 컨텍스트로
else:
    print("⛔ 10회 안에 못 풂 — 가드레일 작동")

여기서 진짜 에이전트로 가는 길은 딱 두 줄을 바꾸는 것뿐입니다: model() 안을 실제 LLM 호출로, verify() 안을 실제 검증(예: 테스트 실행)으로. 나머지 루프 골격은 그대로입니다.
